## 6. Local Supabase Setup (NEW — read this before touching `supabase start`)

`supabase/config.toml` carries three local-dev-only workarounds that took some debugging. They're documented inline in the file but the **why** matters:

### 6.1 Dummy Twilio block — unlocks `test_otp`

```toml
[auth.sms.test_otp]
639171234567 = "123456"

[auth.sms.twilio]
enabled = true
account_sid = "AC_LOCAL_DUMMY"
message_service_sid = "MG_LOCAL_DUMMY"
auth_token = "env(SUPABASE_AUTH_SMS_TWILIO_AUTH_TOKEN)"
```

GoTrue refuses to enable phone login unless **at least one provider is enabled**, even if you only want `test_otp`. Without this block the CLI logs `WARN: no SMS provider is enabled. Disabling phone login` and the `phone:true` flag never makes it into `/auth/v1/settings`. Dummy creds are short-circuited by `test_otp` matches **before** the provider is ever invoked, so we never need a real Twilio for the test phone.

The unset env var warning at startup is benign — `auth_token` is read but never used because no real send happens.

### 6.2 `[analytics] enabled = false` — Colima can't share docker.sock

The `supabase_vector_tdpos-local` container needs to mount the host's docker socket for log collection. With `--mount-type virtiofs` (Colima's default), the Unix socket file doesn't pass through. With `sshfs`, same result. The only real fix is Docker Desktop or disabling the analytics stack. Hosted Supabase keeps analytics on; this disable is local-only.

### 6.3 `[edge_runtime] enabled = false` — Deno crashes on Colima x86_64

The edge runtime container exits with `Bus error (core dumped)` on Colima's x86_64 emulation. Edge functions ship to hosted Supabase, not local — disabling locally avoids the startup failure and we lose nothing for the smoke-test flow.

### 6.5 Restarting from cold

```bash

colima start --cpu 4 --memory 4 --mount-type sshfs

# Then:

```

Colima preserves Docker images and volumes between restarts, so subsequent starts skip the ~10-image pull (~2 GB).

The migrations re-apply, but the seed in `supabase/seed.sql` is currently empty — the owner-specific seed (the auth.uid + business + branch + product) was applied via inline `psql` in this session and **lives only in the volume**. ## 9. Pattern Library (load-bearing — follow when adding code)

### 9.1 Discriminated-union queries (used 11+ times)

```ts
import 'server-only' // web only

export type FooResult =
  | { ready: false; reason: 'supabase_unconfigured' | 'query_failed'; message?: string }
  | { ready: true; ... }

export async function getFoo(): Promise<FooResult> {
  let supabase
  try { supabase = await getServerSupabase() } catch { return { ready: false, reason: 'supabase_unconfigured' } }
  const { data, error } = await supabase.from('...').select('...')
  if (error) return { ready: false, reason: 'query_failed', message: error.message }
  return { ready: true, ... }
}
```

Server Component consumer:

```tsx
const result = await getFoo()
if (!result.ready)
  return result.reason === 'supabase_unconfigured' ? (
    <ConfigureNotice />
  ) : (
    <ErrorTile msg={result.message} />
  )
return <Tile data={result.data} />
```

### 9.2 supabase-js relation arrays — always destructure first element

```ts
interface Row {
  products: Array<{ name; unit_label }> | null
}
const product = row.products?.[0] ?? null
```

### 9.3 Mobile checkout flow

`executeCheckout(params)` is canonical. **Do not write a parallel checkout path.**

1. Validate cart non-empty, payment method picked, tendered ≥ total for cash.
2. Pre-flight stock check (return `insufficient_stock` before mutating).
3. Look up existing sale by `clientOperationId` (idempotency).
4. `db.withTransactionAsync(...)` — single transaction.
5. `createClientOperationId()` for IDs.
6. `generateReceiptNumber()` for receipts.

### 9.4 Auth state machine

- Sign-in → `signInWithOtp({ phone })` → navigate to `verify-otp`.
- Verify → `verifyOtp({ phone, token, type: 'sms' })` → **do nothing else.**
- The auth listener catches `SIGNED_IN`, runs `bootstrapAuthFromSession()`, populates auth-store. `Stack.Protected` auto-flips.
- On bootstrap failure (`account_not_provisioned` etc.), the listener writes the outcome to `auth-store.bootstrapStatus`. Both `(auth)` screens render `<ErrorBanner>` from `describeBootstrapFailure(outcome)`.

```tsx
  message={bootstrapError}

  onAction={signOutAndReset}
/>
```

Use this for **any** error tile inside an (auth) or (app) screen instead of inline Surface + Text.

### 9.6 Parallel supabase-js queries when shapes are independent

```ts
const [a, b] = await Promise.all([
  supabase.from('x').select('...').eq('y', z).maybeSingle(),
  supabase.from('p').select('...').eq('q', z).maybeSingle(),
])
```

`bootstrapAuthFromSession` does this for branches + businesses (both depend only on `business_id` from the users row).

## 10. Pitfalls — Things The Next Agent Will Trip On

### Carry-over from previous handoff (still relevant)

1. **Forbidden-patterns scanner is aggressive** — catches the legacy session-fetch literal, "Official Receipt", "BIR-compliant", "middleware.ts", etc. **even in comments**. Reword anti-pattern descriptions without the literal string. Example: "the deprecated Next 15 entry name" not "middleware.ts".
2. **`docs/skills/deprecations.md` is the single source.** Don't duplicate the deprecations table in CLAUDE/AGENTS/GEMINI/CODEX — they each carry one-line refs only. DocGate-2.
3. **All 22 skill docs** must have `## Sources` + `Last verified: YYYY-MM-DD`. `scripts/check-skill-docs.mjs` enforces. Adding a 23rd needs the same.
4. **Never edit `001_initial_schema.sql` and `LOCAL_SCHEMA_SQL` separately.** They must stay byte-equivalent. Edit both, or — better — add a new migration.
5. **Demo-mode is gated on `__DEV__`** in `app/(auth)/sign-in.tsx` and now coexists with real OTP. Production builds hide it. Don't introduce a non-dev demo path.
6. **supabase-js types embed relations as arrays** — even for single-row FKs. Always `?.[0] ?? null`.
7. **`SupabaseClient.rpc()` returns `PostgrestFilterBuilder`** (thenable, not Promise). Custom interfaces use `PromiseLike`. Cast at integration boundaries with a comment.
8. **Lint `caughtErrorsIgnorePattern` is not configured.** Use bindless `try { ... } catch { ... }` instead of `catch (_err)`.
9. **Modules default OFF** (ADR-008). Gate every utang/loyalty/SMS UI on `useSettingsStore((s) => s.modules.utang)`. Hidden, not disabled.
10. **PII never surfaces in dashboard or diagnostics** (ADR-014).
11. **`.env.local` is gitignored** (`.env.local` pattern). Verify with `git check-ignore` before any commit.
12. **`bun.lock` is text and committed.** Don't regenerate without reason.
13. **Web Server Component date-range** is via URL param. The `parseDateParam()` helper handles invalid input.
14. **`server-only` package** is in `apps/web/package.json`. New web queries should `import 'server-only'`.
15. **EAS project ID** is hardcoded in `app.config.ts` with `EAS_PROJECT_ID` env override.
16. **Auth listener uses `onAuthStateChange` for cold-start** — NOT a direct cached-session read. Forbidden-patterns scanner blocks the legacy path.
17. **`bootstrapAuthFromSession()` failures are surfaced via discriminated union.** UI renders friendly errors via `describeBootstrapFailure(outcome)`.

## 11. Type-Friction Notes (saves rediscovery)

| Friction                                                            | Cause                                                                                                                        | Fix applied                                                                                   |
| ------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| `SQLiteDatabase` not assignable to `AsyncSqliteLike`                | `expo-sqlite` typed params as required, mine had `?`                                                                         | Made params required in `AsyncSqliteLike`                                                     |
| `readonly` array contravariance                                     | Real client wants mutable arrays                                                                                             | Use `(string \| number \| null)[]` (mutable)                                                  |
| `SupabaseClient.rpc` returns thenable not Promise                   | supabase-js v2 `PostgrestFilterBuilder`                                                                                      | All custom interfaces use `PromiseLike`                                                       |
| `functions.invoke` body type narrower than `unknown`                | `FunctionInvokeOptions.body` is union                                                                                        | Cast at integration boundary with comment                                                     |
| `_err` in `catch (_err)` flagged by lint                            | `caughtErrorsIgnorePattern` not configured                                                                                   | Use bindless `catch { ... }`                                                                  |
| Distributed conditional type returned `never`                       | Conditional doesn't distribute over non-naked union                                                                          | Import the discriminated variant directly                                                     |
| `OrderableQuery` extends `MaybeSingleResult` chain (auth-bootstrap) | supabase-js `.eq().order().limit().maybeSingle()` requires extending the eq result with both `.order()` AND `.maybeSingle()` | New nested interface — see [auth-bootstrap.ts](../apps/mobile/src/services/auth-bootstrap.ts) |
| **NEW** Cast `supabase as unknown as SupabaseBootstrapClient`       | The narrow port type doesn't fully model `PostgrestFilterBuilder`'s structural shape                                         | Cast at the listener boundary; documented inline                                              |
