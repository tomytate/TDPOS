## 1. Project Identity And Release Pact

**TD POS** — offline-first POS for Philippine commerce. Pitch: "Tama ang stock mo. Lagi."

Release model the owner has set:

- **Now → 0.1 → 0.9** = lay the full mobile + web + backend scaffold. No surface-by-surface polish in this window.
- **0.9** = the concentrated testing, polish, visual QA, accessibility, and performance pass. Tier-by-tier acceptance.
- **v0.1alpha** = first real pilot store **after** 0.9. Not before.
- **v0.1alpha → v1.0 gold** = App Store + Play Store + marketing site launch, simultaneously.

Core pact (top of [docs/road-to-1.0-enterprise-checklist.md](road-to-1.0-enterprise-checklist.md)):

- No release date. v1.0 is a quality bar.
- No partial release — mobile + web + marketing all clear v1.0 the same day (ADR-010).
- Enterprise-grade from day one — every dependency aligned with official docs; deprecations single-sourced + machine-enforced.
- Inventory correctness wedge (canonical pieces, delta sync, idempotent `client_operation_id`) — non-negotiable.
- BIR-ready language only until accredited. Mechanical scanner enforces.
- Five canonical product tiers (ADR-016). Old six-name tiers are migration-only.
- Scaffold-first 0.1→0.9, then pilot (ADR-017).

Spec entry-point: [docs/spec-v5.md](spec-v5.md). Master checklist: [docs/road-to-1.0-enterprise-checklist.md](road-to-1.0-enterprise-checklist.md).

## 9. Pattern Library

### 9.1 Tier reads always go through helpers

```ts
import { normalizeSubscriptionTier, getTierDefinition, resolveTierModuleState } from '@tdpos/shared'

const tier = normalizeSubscriptionTier(row.subscription_tier) // canonical, never null/legacy
const definition = getTierDefinition(tier)
const modules = resolveTierModuleState(tier, row.module_state) // tier baseline merged with DB overrides
```

### 9.2 Surface gating is read-side

```ts
import { isTierSurfaceEnabled } from '@tdpos/shared'

if (!isTierSurfaceEnabled(currentTier, 'web.products')) {
  // render <TierLockBanner /> on web, <LockedSurfaceCard /> on mobile
}
```

Adding a surface = add to the `TierSurface` union + add to `SURFACE_LABELS` + add to the right `TIER_DEFINITIONS[*].surfaces` arrays. TypeScript exhaustiveness keeps the lists honest.

### 9.3 Server-side enforcement

```ts
import { checkSurfaceAccess, checkInsertLimit } from '@/lib/entitlements/surface-access'

// In a Server Action:
const access = await checkSurfaceAccess('web.products')
if (!access.ok) return { error: access }
const limit = await checkInsertLimit({ table: 'products', limit: 'products' })
if (!limit.ok) return { error: limit }
// proceed with mutation
```

Backed by Postgres RPCs `current_business_can_use_surface` and `assert_business_limit` (defined in `20260510000001_entitlement_guards.sql`).

### 9.4 Mobile entitlement cache (offline gating)

```ts
import { canUseMobileSurfaceFromCache } from '@/services/entitlement-cache'

const access = canUseMobileSurfaceFromCache({
  tier: store.subscriptionTier,
  surface: 'mobile.shift_handoff',
  entitlementsValidUntil: store.entitlementsValidUntil,
})
// access.allowed === false with reason in {'tier_locked','entitlements_stale'} when failing
```

Tier A cashier always returns `allowed: true`. Manager / owner surfaces require `entitlementsValidUntil + 7-day grace > now`.

### 9.5 Discriminated-union queries (web Server Components, used 11+ times)

```ts
import 'server-only'

export type FooResult =
  | { ready: false; reason: 'supabase_unconfigured' | 'query_failed'; message?: string }
  | { ready: true; ... }
```

### 9.6 Pricing copy is single-sourced

`formatTierPrice(definition.pricePhpMonthly)` everywhere — `Free forever` / `Pricing coming soon` / `₱n/month`.

## 10. Pitfalls (carry-over + new)

### From the 2026-05-10 handoff (still relevant)

1. Forbidden-patterns scanner is aggressive (catches the legacy session-fetch literal, "Official Receipt", "BIR-compliant", "middleware.ts", etc. **even in comments**).
2. `docs/skills/deprecations.md` is the **single source** of deprecations. DocGate-2.
3. All 22 skill docs need `## Sources` + `Last verified: YYYY-MM-DD`. DocGate-3 enforces.
4. Never edit `001_initial_schema.sql` and `LOCAL_SCHEMA_SQL` separately. Add a new migration instead.
5. supabase-js types embed relations as arrays. Always `?.[0] ?? null`.
6. `SupabaseClient.rpc()` returns `PostgrestFilterBuilder` (thenable, not Promise). Custom interfaces use `PromiseLike`.
7. Lint `caughtErrorsIgnorePattern` is not configured. Use bindless `try { ... } catch { ... }`.
8. Modules default OFF (ADR-008). Hidden, not disabled.
9. Demo-mode in mobile sign-in is gated on `__DEV__`. Removal is a v1.0 concern, not v0.1alpha.
10. Auth listener uses `onAuthStateChange` for cold-start — never a direct cached-session read. Forbidden-patterns scanner blocks the legacy path.
11. `auth-store.bootstrapStatus` is excluded from MMKV persist via partialize (session-scoped).
12. EAS log files are **not** standard gzip/zstd — read via the dashboard URL, not curl.
13. VPN intercepts both LAN AND ngrok tunnel. Turn off VPN for mobile testing.
